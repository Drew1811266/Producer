# Producer 交互规则

## 交互目标
Producer 的交互不是“给用户一块空白画布自己发明规则”，而是用一套固定、可学习、低摩擦的交互语法，让用户在分层 context graph 中持续推进创作。

## 全局界面规则
- 画布永远是主表面，默认占据工作区最大面积。
- 浮层、顶部信息、底部状态条都只能服务画布，不能把主流程拉回传统后台式布局。
- 页面进入后，用户首先看到的是当前 graph，而不是表单、列表、设置页或素材管理页。
- 本地项目状态优先于远端状态；交互设计必须假设用户可以离线工作。
- 系统一次只激活和渲染当前 context canvas，不把 brief、storyboard、shot lab 同时铺成一张大图。

## 相机与语义缩放
- 画布必须支持地图式平移和缩放，操作感觉应接近地图软件，而不是普通流程图拖拽。
- 缩放后的内容变化应基于屏幕投影尺寸或 LOD，而不是简单的死阈值切换。
- 远景时优先保留节点主标题、主要结构和必要关系。
- 中景时逐步恢复节点摘要、状态和有限缩略图。
- 近景时再显示完整缩略图、更多文本和辅助关系。
- 关系线也应参与语义缩放：全局视图只保留主要结构关系，近景或选中节点时再恢复更多语义边。

## 层与节点白名单
当前模板 `ecommerce_ad_v1` 必须严格执行 layer-specific whitelist。

- `brief` 层只允许 `brief` 节点。
- `storyboard` 层只允许 `storyboard_shot` 节点。
- `shot_lab` 层只允许 `prompt`、`still`、`video`、`reference`、`review`、`result` 节点。

白名单约束必须覆盖创建、粘贴、导入、复制和持久化写入，不能只在 UI 显示层做提醒。

## 核心手势语法
- 单击：选择节点，并打开或聚焦单一抽屉，用于查看和编辑当前节点内容。
- 双击：沿 `child_canvas_for` 定义钻取到下一层画布。双击是“进入上下文”，不是“打开弹窗”。
- `Tab`：在当前画布中快速添加节点，候选项必须由当前 layer 白名单过滤。

这三个动作构成主循环。Producer 不应依赖复杂右键菜单、深层工具栏或多窗口管理来完成日常编辑。

## 抽屉规则
- 抽屉是单一侧边抽屉，不是多栏后台。
- 抽屉只承载当前节点的语义信息编辑，例如文本内容、引用关系、评审结论、资产绑定和必要元数据。
- 抽屉不能演化成独立页面，也不能替代画布成为主要浏览界面。
- 单击打开抽屉后，画布上下文必须仍然可见，避免“编辑即离场”。
- 点击画布空白区域，抽屉自动收回。
- 按 `Esc` 必须关闭当前抽屉。
- 点击另一个节点时，抽屉不应先关闭再重开，而应保持打开并切换内容。
- 抽屉必须是覆盖式浮层，不能通过挤压主画布来腾出空间。
- 当抽屉会遮挡当前选中节点时，画布应自动进行最小量平移，保证该节点仍然可见。

## 钻取规则
- 根入口固定从 `Brief Canvas` 开始。
- 只有被模板声明为可生成子画布的节点，才允许双击钻取。
- 当前模板里，`brief` 可以进入 storyboard，`storyboard_shot` 可以进入 shot lab。
- 没有子画布定义的节点，双击不应触发任意导航，更不能退化成任意层级的无限展开。

## 节点快速创建器
- 只有当画布区域拥有键盘焦点时，`Tab` 才应弹出快速创建器。
- 快速创建器应出现在鼠标附近，而不是屏幕中央。
- 快速创建器必须自动聚焦搜索输入框，优先支持键盘筛选。
- 菜单项必须严格受当前 layer 白名单约束，系统锚点不能出现在创建器中。
- 按 `Enter` 创建节点，按 `Esc` 关闭创建器。
- 新节点默认创建在当前鼠标所在的世界坐标，而不是画布中心。
- 新节点创建后应自动选中，并自动打开节点内容抽屉，便于立即补全内容。

## 关系规则
Producer 的边是语义关系，不是数据流连线。

- 允许的默认关系包括 `contains`、`references`、`approved_from`、`variant_of`、`alternative_to`、`reuses`。
- 关系用于表达来源、包含、复用、变体、替代和确认结果。
- 关系线不能表达“先执行谁、后执行谁”“把 A 输出给 B”“把参数传给下一个节点”。
- 视觉呈现上，关系应服务理解与追溯，而不是模拟节点编程界面。

## 画布行为规则
- 节点布局服务于理解上下文，不服务于任意涂鸦。
- 视觉自由度可以存在，但不能破坏层级语义与可读性。
- 同一画布里的操作应围绕选择、连接、添加、整理和钻取，而不是加入大量与语义无关的装饰工具。
- 同一时刻用户操作的对象应始终属于当前 graph，不允许通过拖入模板、跨层粘贴或导入捷径绕过层边界。
- 当前实现尚未完整落地相机、节点渲染和抽屉逻辑，但后续实现必须服从这些规则，而不是临时为工程方便改变交互含义。

## 明确禁止
- 不做通用白板式自由图形系统。
- 不做在线协作导向的评论流和多人 presence 主界面。
- 不做看板式任务编排替代主画布。
- 不做素材库优先的媒体管理器界面。
- 不做 Houdini 式 executable node graph，不引入算子求值或数据管线语义。
