# Producer 产品愿景

## 产品定义
Producer 是一个面向 AI 时代视觉创作者的本地优先桌面工作台。它的核心不是“画东西”，而是把 Brief、Storyboard、Shot Lab 这类创作上下文组织成可钻取、可追溯、可扩展的分层语义图，让创作者始终在同一张画布体系内推进项目。

Producer 的第一性原理是：创作不是线性的表单填写，也不是一堆文件夹和聊天记录的堆叠，而是围绕目标、镜头、变体、参考、评审结果不断分叉与收敛的上下文网络。

## 三层上下文图
当前产品定义以 `ecommerce_ad_v1` 为首个模板，明确采用分层 context graph，而不是单层无限画布。

- `Brief Canvas`：项目根画布，只允许 `brief` 节点，负责沉淀目标、受众、卖点、风格与约束。
- `Storyboard Canvas`：从 brief 派生，只允许 `storyboard_shot` 节点，负责把策略拆成镜头级结构。
- `Shot Lab Canvas`：从 storyboard shot 派生，只允许 `prompt`、`still`、`video`、`reference`、`review`、`result` 节点，负责生成实验、对照与收敛。

这三层不是任意展开的树状文件夹，而是带有明确语义边界的 graph-of-graphs。每一层都只承载该层应有的信息密度与决策粒度。

## 核心原则
- 本地优先：项目、数据库、资产、导出都首先存在于本地，Producer 先保证个人工作流完整，再考虑外部连接。
- 画布优先：画布是主界面，其他控件必须退居覆盖层或抽屉，不能反客为主。
- 当前上下文优先：系统一次只激活和渲染当前 graph，不把所有层摊平到一张无限大图上。
- 地图式浏览：画布必须支持大范围、丝滑的平移和缩放，用户可以像看地图一样在全局与局部之间切换。
- 语义缩放：缩小时逐步隐藏缩略图、次要文案与辅助关系，只保留主结构；放大后再恢复局部细节。
- 严格分层：节点类型按 layer 白名单约束，不能把任何内容随意丢进任意画布。
- 语义关系优先：边连接的是 `contains`、`references`、`approved_from`、`variant_of`、`alternative_to`、`reuses` 这类意义关系，而不是执行顺序或数据流。
- 轻界面高频操作：单击、双击、Tab 是核心操作语法，减少模式切换和重面板操作。
- 模板先行：先用一个垂直模板跑通闭环，再复制到更多创作场景，而不是先做一个空泛平台。

## 明确边界
Producer 不是通用白板。它不追求“什么都能画”，而是只接受与创作上下文有关的结构化节点和关系。

Producer 不是在线协作平台。它当前不以多人实时协同、评论流、presence 或云端同步为核心卖点。

Producer 不是看板或素材库。任务状态管理和素材收纳可以作为辅助手段存在，但不能成为主模型，更不能压过画布与上下文图。

Producer 不是 Houdini 式可执行节点系统。节点之间表达的是创作语义和来源关系，不是算子连接、参数求值或可执行数据流。

Producer 也不是“把所有内容装进一张超级大图”的系统。它是 graph-of-graphs：Brief 管 Brief，Storyboard 管 Storyboard，Shot Lab 管镜头实验，每次只处理当前语境。

## 当前实现基线
当前仓库已经落地的是 Phase 0 + Module A 的启动基线，而不是完整产品。

- 已有最小全屏 workspace shell，画布是主表面，辅助信息在浮层中。
- 已有本地项目创建与打开流程，项目目录会初始化 `.producer/`、`assets/`、`exports/`。
- 已加载声明式模板 `ecommerce_ad_v1`，并对 layer 与 node 白名单做严格校验。
- 当前只会 seed 根 `Brief Canvas`，不会预先生成 storyboard 或 shot lab 图。
- 根画布进入后仍是占位态，说明产品目前确认了空间结构和边界，但还没有完成完整交互闭环。

## V1 北极星
V1 不是把 Producer 做成一个“大而全创作平台”，而是把一条清晰的工作流跑通：从根 `Brief Canvas` 出发，进入 storyboard，再进入 shot lab，在本地完成创意结构化、生成实验、结果沉淀与回溯。

如果这个闭环成立，Producer 才有资格继续扩展更多模板、模块和自动化能力。任何后续扩展都必须复用同一套分层语义图与交互语法，而不是把产品拉回通用白板或执行流编辑器。
